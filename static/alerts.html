Without your node key, your report card covers agreement scores and version — about 45% of the full picture. With it, you get latency, uptime, peer count, and geographic diversity scoring. Your composite score becomes more accurate, and you help improve the network diversity map for the whole community.
  1. SSH into your validator server:
    ssh root@YOUR_SERVER_IP
  2. Run this command to get your node key:
    docker exec postfiatd curl -s -X POST http://127.0.0.1:5005 -H 'Content-Type: application/json' -d '{"method":"server_info","params":[{}]}' | python3 -c "import sys,json; print(json.load(sys.stdin)['result']['info']['pubkey_node'])"

    This queries your node's RPC and prints just the node public key.

  3. The output will look like this — a key starting with n9:
    n9KbsDV8xeD9XaKZxEasvGMFGdhmH3bBaSBmFvDzVZQzBVfAWcFv
  4. Paste it into the Node Public Key field above and submit.
Is this safe? Your node key is your node's network identity — it's not secret. It's visible to every peer your node connects to. Sharing it here just lets us link your topology data (IP, latency, uptime) to your validator scores for a more accurate composite score.

How to Create a Discord Webhook

  1. Open Discord and right-click your server name at the top of the channel list. Select Server Settings.
  2. In the left sidebar under Apps, click Integrations.
  3. Click Webhooks, then click New Webhook.
  4. Give it a name (e.g. "Validator Alerts"), and use the Channel dropdown to pick which channel receives alerts.
  5. Click Copy Webhook URL — paste it into the form above.

Webhook URLs look like: https://discord.com/api/webhooks/123.../abc...
Note: Do NOT use the gear icon at the bottom-left — that opens User Settings, not Server Settings.
